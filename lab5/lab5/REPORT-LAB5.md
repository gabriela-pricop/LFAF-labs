## Theory

A **context-free grammar (CFG)** is defined as a 4-tuple G = (V_N, V_T, P, S), where V_N is the set of nonterminal symbols, V_T is the set of terminal symbols, P is the set of productions, and S ∈ V_N is the start symbol.

A grammar is in **Chomsky Normal Form (CNF)** if every production has exactly one of the following forms:

- **A → BC** where A, B, C ∈ V_N (binary rule over nonterminals)
- **A → a** where A ∈ V_N and a ∈ V_T (single terminal rule)
- **S → ε** only if ε belongs to the language generated by G (epsilon rule for start symbol only)

CNF is significant because it is the standard normal form used by the **CYK (Cocke–Younger–Kasami) algorithm** for parsing context-free languages in O(n³) time.

### Normalization Steps

Converting an arbitrary CFG to CNF requires the following transformations in order:

1. **Eliminate ε-productions** — remove all rules of the form A → ε (except for S if ε ∈ L(G)).
2. **Eliminate unit (renaming) productions** — remove rules of the form A → B where both A and B are nonterminals.
3. **Eliminate inaccessible symbols** — remove all nonterminals (and their productions) that cannot be reached from the start symbol.
4. **Eliminate nonproductive symbols** — remove all nonterminals that cannot derive any string of terminals.
5. **Convert remaining productions to CNF** — replace terminals in long rules with new nonterminals, and binarize rules of length ≥ 3.

Every transformation method returns a **new `Grammar` instance** (immutable pipeline), leaving the original unchanged. A `Copy()` helper performs a deep clone for safe mutation inside each step.

### Step 1 — Eliminate ε-productions

The algorithm first computes the **nullable set** N_ε: the set of nonterminals that can derive ε. It starts with any A for which A → ε exists, then adds any nonterminal whose every production consists entirely of nullable symbols:

```csharp
bool changed = true;
while (changed)
{
    changed = false;
    foreach (var (head, rhsList) in g.Productions)
        foreach (var rhs in rhsList)
            if (rhs.Count == 0 || rhs.All(s => nullable.Contains(s)))
            { nullable.Add(head); changed = true; break; }
}
```

Then, for each production A → α, all `2^k` subsets of positions (where k is the number of nullable symbols in α) are generated via a recursive backtracking function `NullableExpansions`. The empty production is only kept if the head is the start symbol.

### Step 2 — Eliminate Unit Productions

For each nonterminal A, the **unit closure** U(A) is computed: the set of all nonterminals reachable from A via chains of unit rules A → B → C → ... This uses a standard DFS/BFS:

```csharp
private static HashSet<string> UnitClosure(string start, Grammar g)
{
    var closure = new HashSet<string> { start };
    var stack = new Stack<string>(); stack.Push(start);
    while (stack.Count > 0)
    {
        var head = stack.Pop();
        foreach (var rhs in g.Productions[head])
            if (rhs.Count == 1 && g.VN.Contains(rhs[0]) && closure.Add(rhs[0]))
                stack.Push(rhs[0]);
    }
    return closure;
}
```

For each A, the new productions of A are all non-unit rules of every member of U(A), with duplicates removed.

### Step 3 — Eliminate Inaccessible Symbols

Starting from the set `{S}`, the algorithm repeatedly adds any nonterminal that appears in the right-hand side of a reachable nonterminal's production, until convergence. All nonterminals not in the reachable set, along with their productions, are dropped.

### Step 4 — Eliminate Nonproductive Symbols

A nonterminal is **productive** if it can derive at least one string of terminals. The algorithm seeds the productive set with any NT whose production is a pure terminal string, then iterates: any NT whose production consists entirely of productive NTs and terminals is added, until no more changes occur. Non-productive NTs and any production referencing them are removed.

### Step 5 — Convert to CNF

CNF conversion proceeds in two phases:

**Phase 1 — Terminal isolation:** For every production of length ≥ 2, each terminal symbol `a` is replaced by a fresh nonterminal `T_a`, with the added rule `T_a → a`.

**Phase 2 — Binarization:** Every production of length ≥ 3 is split into cascaded binary productions by introducing fresh nonterminals `X`, `X1`, ..., For example:
```
A → B C D E  becomes  A → B X1,  X1 → C X2,  X2 → D E
```

A pair-memoization map avoids creating duplicate helper nonterminals for the same pair of symbols.

### CNF Validation

The `ValidateCnf()` method verifies the final grammar against all three CNF constraints, returning a boolean flag and a list of any violations found.

### Step 1 — After Eliminating ε-productions

N_ε = {C} (C → ε, and no other NT has all-nullable productions).

Productions added by removing C from right-hand sides:
- B → AC gains B → A (since C is nullable)
- S → AC gains S → A

Result:

```
P = {
  A → d | dS | aBdB
  B → a | aA | A | AC
  C → b | bC          (ε removed, bC with C possibly absent → b)
  D → AB
  S → dB | A | AC
}
```

### Step 2 — After Eliminating Unit Productions

Unit rules present: B → A, S → A.

Unit closures:
- U(B) = {B, A}  → B inherits all non-unit rules of A
- U(S) = {S, A}  → S inherits all non-unit rules of A

Result (D removed as inaccessible in next step, C is nonproductive in step 3):

```
P = {
  A → d | dS | aBdB
  B → a | aA | AC | d | dS | aBdB
  C → b | bC
  D → AB
  S → dB | AC | d | dS | aBdB
}
```

### Step 3 — After Eliminating Inaccessible Symbols

Starting from S, reachable symbols: S → {d,B,A,C,a,b} so V_N reachable = {S, A, B, C}.
D is never referenced from S or any reachable NT → **D is removed**.

```
V_N = {S, A, B, C}
P = {
  A → d | dS | aBdB
  B → a | aA | AC | d | dS | aBdB
  C → b | bC
  S → dB | AC | d | dS | aBdB
}
```

### Step 4 — After Eliminating Nonproductive Symbols

All of S, A, B, C can derive terminal strings → all are productive.
No productions are removed in this step.

### Step 5 — Chomsky Normal Form

New nonterminals introduced: T_a → a, T_b → b, T_d → d, X, X1, X2, X3.

The production `aBdB` (length 4) is binarized as:
```
A → T_a X3      X3 → B X1      X1 → T_d B
```

Full CNF grammar:

```

V_T = {a, b, d}
P = {
```

## Conclusions

This laboratory work demonstrated the complete normalization pipeline for context-free grammars, resulting in an equivalent Chomsky Normal Form representation. The key results achieved are:

The implementation in C# encapsulates all five normalization steps inside a single `Grammar` class, with each step returning a new immutable grammar object. This makes the pipeline composable and easy to test at each intermediate stage.

The most algorithmically interesting step was **ε-elimination**: generating all 2^k subsets of a production for k nullable positions required a recursive backtracking approach that is generic for any production length and any nullable set.

The **unit closure** computation using DFS ensures that chains of renaming rules A → B → C → ... are correctly collapsed in a single pass, without requiring multiple iterations over the grammar.

For **CNF conversion**, the key insight was separating terminal isolation (Phase 1) from binarization (Phase 2), and using a pair-memoization map to avoid creating redundant binary nonterminals for the same symbol pair.

The bonus generalization was verified against the Python reference implementation (Variant 20), producing matching intermediate grammars and a valid CNF result, confirming the correctness of the approach across different grammars.
